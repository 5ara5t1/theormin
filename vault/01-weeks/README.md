# Weekly files

One file per week. It contains everything for that week: plan, sessions, problems, reading checked off, theorems reproved, scans link, reflection.

## Convention

- `week-01.md`, `week-02.md`, etc.
- Frontmatter: `type: week`, `week: N`, `start_date`, `end_date`, `phase`.
- Sections required for the dashboard to work: `## Sessions` (table) and `## Problems worked` (table). Both tables are read by `scripts/extract_from_vault.py`.

## What goes where

- **Sessions table** — one row per study block. `Date | Hours | Subject | Activity`.
- **Problems table** — one row per problem worked. `Date | Book | Ref | Subject | Min | Status | Note`.
- **Reading completed** — checklist of chapters read this week.
- **Chapter summaries written** — checklist with links to the summary notes in `02-summaries/`.
- **Theorems reproved** — short list with links to `03-theorems/` notes.
- **Scans** — link to `05-derivations/week-XX/` containing the PDF dump.
- **Reflection** — written at end of week.
